## Why

PokerMap currently ships with six placeholder demo clubs that all carry the disclaimer "Демо-данные. Замените реальной информацией." and lack any real telephones, social links, or accurate hours. The public map therefore has no real value yet. At the same time three concrete UX problems block usefulness even if the data were real: (1) operators edit the weekly schedule as raw JSON in a `<textarea>` inside the admin form, so it is impossible to use without manual JSON authoring; (2) the public list (`/list`) renders club cards that are not clickable and no `/clubs/[slug]` page exists, so users cannot see full details; (3) the map renders every published marker with no filters, so users cannot answer "what's open now" or "where can I play PLO right now". This change ships real Saint Petersburg poker club data, replaces the JSON working-hours editor with a typed UI editor, adds a public club detail page reachable from the list and the map sheet, and adds map-level filters for open-now / game type / buy-in.

## What Changes

- **Catalog data**: Seed real SPB poker clubs sourced from public listings (PokerDiscover, 2GIS, club websites and Telegram channels) including Bluff (Фурштатская 44), Pulse (Бугский пер. 3), Эло клуб (Каменноостровский 10Б), Saint-Pokersburg (Невский пр.), O'Briens (Невский пр.), Cromulent Club (Владимирский пр. 15), Joker Club SPB, Pokeroff, Royal Club. Each entry carries address, coordinates, phones, website, Telegram URL, working hours, game list, rake description and buy-in bounds. Existing demo rows in `internal/seed/seed.go` are replaced (not appended) and a one-shot data migration prunes any seeded demo slugs already present in the database.
- **DB schema**: Extend `clubs` with a typed `club_type` column (cash / club / mtt-series / mafia-and-poker / underground, default `cash`), a `social_links` JSONB column (Instagram / VK / YouTube / Telegram channel separately from the existing `telegram_url`) and an `entry_fee_cents` nullable column for venues that charge a flat entry. Working-hours JSON shape is formalized into a typed schema validated server-side (`{ mon..sun: { closed: bool, slots: [{ open: "HH:MM", close: "HH:MM" }] } }`) and rejected at the API boundary instead of accepted as opaque JSON.
- **Admin working-hours UI**: Replace the JSON `<textarea>` in `apps/admin/src/app/(authed)/clubs/_form/ClubForm.tsx` with a `WorkingHoursEditor` component that renders one row per weekday with a "Closed / Open" toggle, time pickers for open/close, "Add slot" for split shifts, "Copy to all days" and "Copy Mon–Fri" actions. The editor serializes to the typed JSON schema for submission.
- **Admin club-type and socials**: Add fields in the same form for `club_type` (select), `entry_fee_cents` (number), and a `socials` group with one input per platform (vk, instagram, youtube, telegram channel) instead of one free-text URL.
- **Public club detail page**: New route `apps/web/src/app/clubs/[slug]/page.tsx` that fetches `/api/v1/clubs/{slug}` server-side and renders name, address, phones with `tel:` links, socials, website, working-hours table with today highlighted, games, buy-in, rake, photos. The list cards (`ListClient.tsx`) and the map's `ClubSheet` get a "Подробнее" link that navigates to `/clubs/[slug]`.
- **"Open in maps" deep link**: Every place a club address is shown (detail page, list card, marker sheet) gets a primary "Открыть в картах" button. On iOS it deep-links to Apple Maps (`maps://?daddr=lat,lng&q=<address>`) with a Yandex Maps (`yandexmaps://maps.yandex.ru/?pt=lng,lat&z=17`) fallback if Apple Maps URL handler is absent; on Android it uses the `geo:lat,lng?q=<address>` intent so the system chooser surfaces Google Maps / 2GIS / Yandex; on desktop it opens `https://yandex.ru/maps/?pt=lng,lat&z=17` in a new tab. Selection is done client-side from `navigator.userAgent` / `navigator.platform` plus a runtime check for the Telegram WebApp host (where it must use `Telegram.WebApp.openLink` to escape the in-app browser). A small dropdown menu next to the button exposes "Яндекс.Карты", "2ГИС", "Google Maps" as explicit choices for users who don't want the platform default.
- **Map filters**: New floating filter pill on the map (mobile bottom-left, desktop top-left) opens a `Sheet` containing: "Открыто сейчас" toggle, multi-select chips for games (NLH, PLO, PLO5, MTT, SnG, Mixed), multi-select chips for club type, and a min/max buy-in slider. Active filters are reflected in the URL (`?openNow=1&games=NLH,PLO&types=cash`), survive reload, are shareable, and reduce both the markers on the map and the list view in `/list`. A small badge on the pill shows the active filter count and a "Сбросить" action clears them.
- **Open-now evaluation**: A shared `isOpenNow(workingHours, now, timezone)` helper in `@pokermap/types` (or a new `@pokermap/clubs` utility) computes whether a club is open at the current Europe/Moscow time, correctly handling overnight slots that cross midnight (e.g. `18:00–06:00`).

### Modified Capabilities
- `club-catalog`: Adds requirements for the real-club seed data, the typed working-hours schema, the new `club_type` / `social_links` / `entry_fee_cents` fields, and the `isOpenNow` semantics that the filter relies on.
- `admin-panel`: Adds requirements for the structured `WorkingHoursEditor` UI, the per-platform socials inputs, and the `club_type` / `entry_fee_cents` fields in the club form.
- `public-map-ui`: Adds requirements for the on-map filter pill (open-now, game type, club type, buy-in), URL persistence of active filters, the public `/clubs/[slug]` detail page, the "Подробнее" link from both the list cards and the marker sheet, and the platform-aware "Открыть в картах" deep link (Apple Maps / `geo:` / Yandex / 2ГИС / Google Maps) including the Telegram-WebApp `openLink` path.

## Impact

- **Schema / data**: New migration `0004_clubs_typed_hours_and_filters.up.sql` adds `club_type` enum, `social_links jsonb NOT NULL DEFAULT '{}'`, `entry_fee_cents bigint`, and a CHECK / trigger that validates `working_hours` against the typed shape. Down migration drops the columns and reverts the check. Seed file `internal/seed/seed.go` is rewritten with real SPB clubs.
- **API (`apps/api/internal/clubs`)**: `dto.go` gains typed `WorkingHours` (struct, not `json.RawMessage`), `ClubType`, `SocialLinks`, `EntryFeeCents`. `validate` package gains a `workingHours` validator. `handler.go` `list` accepts `?openNow=1&games=NLH,PLO&types=cash&minBuyIn=...&maxBuyIn=...` query params and filters at SQL level (open-now applied in-process because Postgres needs the current MSK time to evaluate slot windows). `sqlc` queries get a new `ListPublishedClubsFiltered` query.
- **Shared packages**: `packages/types` exports the new `WorkingHours` shape, `ClubType` union, `SocialLinks` shape, and a pure `isOpenNow` function with timezone parameter and exhaustive tests for overnight slots, midnight wrap, and closed days.
- **Admin app**: New `WorkingHoursEditor.tsx` and `SocialsEditor.tsx` under `_form/`, edits to `ClubForm.tsx` and `actions.ts` to serialize/parse the typed shape, new shadcn primitives (`Switch`, `Slider`, `Sheet` already present) added to `@pokermap/ui` if missing.
- **Web app**: New `app/clubs/[slug]/page.tsx`, new `components/map/MapFilters.tsx`, `useClubFilters` hook that reads/writes URL params and feeds both `MapView` and `ListClient`. `MapShell` mounts the filter pill alongside the existing nav. `ListClient` renders cards as `<Link href={`/clubs/${slug}`}>` and reuses the same filter hook. A shared `components/OpenInMapsButton.tsx` encapsulates the per-platform deep-link logic so the detail page, list card and `ClubSheet` all use the same button.
- **Tests**: Go unit tests for the working-hours validator and the open-now SQL/process filter. Vitest tests for `isOpenNow`, the `WorkingHoursEditor` round-trip, and the `useClubFilters` URL sync. A Playwright spec asserts that toggling "Открыто сейчас" reduces the visible markers and that clicking a list card navigates to `/clubs/[slug]`.
- **Risk**: The DB migration changes the implicit `working_hours` contract. Existing rows whose JSON does not match the new shape would fail the CHECK; the migration normalizes existing seed rows in the same transaction and the demo data is replaced wholesale, so no production data is at risk. There is no public production deployment yet.
